# 让 AI 替我管域名：一次人机协作的实操记录

> 我提需求，AI 执行，我做决策。半天时间，不花一分钱，搭了一套域名管理系统。但真正值得记录的不是结果，而是过程中学到的东西。

---

## 起因

我手里有几个域名。买的时候热血沸腾，觉得"这个以后肯定用得上"。结果呢？大部分都在吃灰。

`.com` 一年几十块，看起来不多。但架不住数量多——等你回过神来，每年续费已经在默默烧钱了。

这次我想换个思路：找免费的子域名服务，注册一批，管好解析，自动化续期提醒。

思路不复杂，执行全是细节：API 调试、密钥管理、DNS 配置、自动化脚本、安全加固。这些活儿有个共同特征——**规则明确，重复性高，适合 AI 做**。

于是我把这个任务扔给了 OpenClaw Agent。

---

## 方案：两个免费工具，各司其职

整个系统用两个服务搭成，一个管注册，一个管解析。

**DNSHE** 负责子域名的注册、续期和生命周期管理。它提供免费子域名，根域名覆盖好几个区域，有效期最长十年。管的是"这个域名归你"。

**Cloudflare** 负责 DNS 解析、CDN 加速和安全防护。免费方案就支持全部常见记录类型，对个人项目来说是天花板级别。管的是"别人怎么找到你"。

这两层解耦的好处很简单：以后想换解析服务商，DNSHE 不用动；DNSHE 的免费政策变了，Cloudflare 的解析记录也不受影响。

各司其职，互不干扰。

---

## 我提需求，AI 做执行

### 第一步：让 Agent 验证 API 连通

我把 Cloudflare 的 API Token 和 DNSHE 的 API Key 交给 Agent，让它自己写 curl 命令验证。它很快跑通了 Cloudflare，但在 DNSHE 上卡住了——直接返回 401。

我只给了 API Key，没给 API Secret。Agent 翻了半天 API 文档，怀疑是 URL 写错了、权限不够、参数格式有问题。折腾了好一会儿，它才注意到文档里写着：**API Key + API Secret，双密钥认证。**

这个教训很朴素：双密钥认证的服务，两个都得给。单给一个不会报"缺少 Secret"，而是直接 401——这个错误信息太暧昧了，容易把排查方向带偏。

补上 Secret，Agent 一口气注册了几个子域名。

### 第二步：让 Agent 封装自动化 Skill

光有 API 还不够。每次操作都翻文档、拼 curl，效率太低。

我让 Agent 把 DNSHE 的常用操作封装成一个管理 Skill：

- 自动续期——检查到期时间，单个或批量续期
- DNS 记录管理——A/AAAA/CNAME/MX/TXT 增删改查
- 注册新子域名——检查可用根域名，注册后自动配 DNS
- Cloudflare 联动——DNSHE 和 Cloudflare 之间的同步脚本

以后再需要管理域名，直接跟 Agent 说一声就行，不用记 API 地址和参数了。

### 第三步：续期提醒

免费域名最大的风险就是忘了续期。我让 Agent 设了定时提醒：有效期短的域名在到期前 180 天通知，有效期长的也按同样标准来。到时间了自动推送，不用自己惦记。

---

## 踩的最大的坑：密钥安全

这可能是整个实践中最重要的教训。

Agent 最初把 API Key 和 Token 明文写在了一个工作文档里。这个文件每次对话都会被加载到 AI 的上下文里——**等于把钥匙挂在门上，谁都能看见。**

我发现了这个问题，立刻让 Agent 整改。四步走：

1. 新建 `.env` 文件，集中存放所有密钥，不进任何对话上下文
2. 新建 `.gitignore`，排除 `.env`，防止误提交到 git
3. 所有文档中的密钥替换为环境变量引用
4. 全盘扫描，清理残留的明文 key

**密钥不该出现在任何会被加载到 AI 对话上下文的文件里**——这是这次踩坑后最深的一条体会。`.env` + `.gitignore` 是最轻量且有效的方案：密钥只存在于本地文件，不进 git，不进 AI 上下文。以后用任何需要 API Key 的服务，第一时间就做这个安全设置。别等出了问题再补。

---

## 回头来看

整套方案的成本是零——DNSHE 免费注册，Cloudflare 免费解析，脚本自己写，续期提醒用 Cron。

但比成本更重要的是过程中暴露的几个问题：

**AI 会犯错，但错误是有意义的。** 401 那个坑，如果是我手动操作，可能也会踩。但 Agent 翻文档排查的过程让我意识到，错误信息的设计是个大问题——一个模糊的 401 能把排查方向带偏半天。

**AI 会犯"习惯性错误"。** 把密钥写进文档这件事，Agent 并不是故意的——它只是习惯性地把所有信息记录在最容易找到的地方。但这个习惯在安全场景下是致命的。人类有安全直觉，AI 没有。所以安全判断这件事，必须人来做。

**自动化不是目的，是手段。** 封装 Skill、设续期提醒，这些不是为了让系统更复杂，而是为了让以后的操作更简单。今天多花半小时写脚本，以后每次操作省十分钟。这笔账算得过来。

最关键的体会是四个字：**人机分工。**

架构设计、安全判断、取舍权衡——这是人的活儿。curl 命令、脚本编写、批量操作——这是 AI 的活儿。谁也别抢谁的活儿。
